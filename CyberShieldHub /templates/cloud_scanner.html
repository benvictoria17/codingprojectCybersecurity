Cloud Services Security Scanner

What is a Cloud Services Scanner?

Cloud services are programs and storage that run on computers far away that you access through the internet. A cloud scanner checks if your cloud services have security problems.

This tool will check common cloud services for security issues.

Cloud Services Scanner

What We Check For

Storage Security

For cloud storage services like S3 buckets and Blob Storage.

  • Public access settings
  • Proper access controls
  • Encryption settings
  • Logging and monitoring
  • Versioning configuration

Access Control

For identity and access management services.

  • Over-privileged accounts
  • Password policies
  • Multi-factor authentication
  • Service account security
  • Unused credentials

Compute Security

For virtual machines and compute services.

  • Security group settings
  • Public IP exposure
  • Operating system updates
  • Disk encryption
  • Network configuration

Database Security

For cloud database services.

  • Public access settings
  • Encryption configuration
  • Backup policies
  • Authentication methods
  • Network security

Cloud Security Tips

Use The Principle of Least Privilege

Give users and programs only the permissions they need to do their job, nothing more.

Enable Multi-Factor Authentication

Require a second verification step (like a text message code) for signing in to cloud accounts.

Always Encrypt Sensitive Data

Use encryption for all sensitive data stored in the cloud, especially personal information.

Enable Monitoring and Logging

Turn on security logging so you can see who accessed your cloud resources and what they did.

Example: S3 Bucket Security Issues

Amazon S3 buckets are often misconfigured. Here's what can go wrong:

S3 Bucket with Dangerous Settings:
{
  "BucketName": "company-data",
  "PublicAccessBlock": false,
  "ObjectACL": "public-read",
  "Versioning": "Disabled",
  "ServerSideEncryption": "Disabled",
  "AccessLogs": "Disabled"
}

Problems:

  • Public access is not blocked
  • Objects can be read by anyone
  • Versioning is turned off
  • Encryption is not enabled
  • Access logging is off
Secure S3 Bucket Configuration:
{
  "BucketName": "company-data-secure",
  "PublicAccessBlock": true,
  "ObjectACL": "private",
  "Versioning": "Enabled",
  "ServerSideEncryption": "AES-256",
  "AccessLogs": "Enabled"
}

Good practices:

  • Public access is blocked
  • Objects are private by default
  • Versioning is enabled to recover from accidental changes
  • Encryption is enabled to protect data
  • Access logging is on to track who accessed what
