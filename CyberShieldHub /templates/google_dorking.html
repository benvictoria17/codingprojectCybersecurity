Google Dorking Educational Tool

What is Google Dorking?

Google dorking means using special search commands to find information that regular searches might not show. Hackers use this to find security problems, but security experts use it to protect websites.

This tool teaches you how Google dorking works and how to protect your information.

Google Dorking Tool

For example, with "site:" dork type, enter a domain like "example.com"

Common Google Dork Commands

Command What It Does Example
site: Limits search to a specific website site:example.com password
filetype: Finds specific file types filetype:pdf confidential
inurl: Finds pages with specific text in the URL inurl:admin
intitle: Finds pages with specific text in the title intitle:"index of"
intext: Finds pages with specific text in the content intext:"internal use only"
- (minus) Excludes pages with specific words password -login
"text" Finds exact phrase matches "confidential document"

How Hackers Use Google Dorking

Hackers use Google dorking to find security problems in websites. Here are some examples:

Finding Sensitive Files

Hackers search for sensitive file types that might contain passwords or private information.

filetype:xls password
filetype:pdf "confidential"
site:example.com filetype:txt password
Finding Database Errors

Hackers look for error messages that show database problems.

intext:"sql syntax near"
intext:"error occurred in database"
intitle:"index of" intext:mysql.conf
Finding Login Pages

Hackers search for login pages to try to break into.

inurl:admin intitle:login
inurl:login.php
intitle:"login page" "username"
Finding Open Cloud Storage

Hackers look for cloud storage that isn't protected properly.

intitle:"index of" "aws-config"
intext:"Index of" "s3://bucket-name/"
site:s3.amazonaws.com filetype:xls

How to Protect Against Google Dorking

Use these tips to protect your website and information from Google dorking:

Use Robots.txt

Create a robots.txt file to tell search engines what not to index.

# Example robots.txt
User-agent: *
Disallow: /admin/
Disallow: /private/
Disallow: /config/
Disallow: /backup/
Password Protect Sensitive Areas

Use password protection for admin areas and private files.

<!-- Example .htaccess file for Apache -->
AuthType Basic
AuthName "Restricted Area"
AuthUserFile /path/to/.htpasswd
Require valid-user
Check Your Own Website

Use Google dorking on your own website to find problems before hackers do.

site:yourwebsite.com filetype:pdf
site:yourwebsite.com inurl:admin
site:yourwebsite.com "password" OR "username"
Secure Cloud Storage

Make sure your cloud storage is not publicly accessible unless necessary.

// Example AWS S3 bucket policy
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
